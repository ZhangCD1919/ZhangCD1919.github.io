# CSS预处理器

# 诞生背景

css 层叠样式表(Cascading Style Sheets)是一种用来表现HTML（标准通用标记语言的一个应用）或XML（标准通用标记语言的一个子集）等文件样式的计算机描述型静态语言。
css诞生之后，基本语法和核心机制一直没有本质上的改变，缺少编程语言的灵活性。CSS 只能用静态的语法描述元素的样式，无法像写 JavaScript 那样增加逻辑判断与共享变量。这与如今模块化、工程化的前端发展方向相悖，在搭建大型项目时较为冗余繁杂。因此，css预处理器应运而生。他们支持你使用变量、函数、运算等'超能力'创建出css，并可运用模块化的依赖和引用。
css预处理器语言无法直接在浏览器运行，最终会编译成css代码使用。

# CSS预处理器对比

目前最为普遍的三款stylus、less、sass，每种预处理器都包含了变量、混写（mixins）、嵌套和扩展，以及不同的度量单位、逻辑运算符和函数。通过这些你可以抽象出许多关键部分，并重复使用，良好的逻辑和更少的代码能让你从能更快更好的实现样式。

# less

优点：

  • 有基本的功能和健全优质的社区及资源
  • 简单易于上手
  • node和js都能编译，不需要额外的编译工具

缺点：

  • 逻辑处理特性少，逻辑能力弱

# sass

优点：

  • 有基本的功能和健全的社区环境
  • 强大的逻辑特性，函数式编程css

缺点：

  • 需要额外的编译工具Ruby

# stylus

优点：

  • 有基本的功能和健全的社区环境
  • 超强大的逻辑特性，函数式编程css
  • 精简、灵活的语法

缺点：

  • 无

# stylus 有多强？

# 语法

less只能这样

/* style.less */
h1 {
  color: #0982C1;
}

sass还可以这样

/* style.scss */
h1 
  color: #0982C1

stylus 除了以上两种，还可以这样，花括号、冒号、分号，省略什么都可以

/* style.styl */ 
h1
  color #0982C1

h1
  color: #0982C1;

# 变量、函数、混写、流程控制

less变量只能用@开头，并且没有流程控制和自定义函数，混写需要常规css格式

/* style.less */
@mainColor: #0982c1;

sass变量只能用$开头，混写必须以@mixin开头，并且使用的时候要用@include，条件控制用@if和@else,函数要以@function开头，并在函数内部有一行@return

/* style.sass */
$mainColor: #0982c1;

stylus变量可以用任意非@字符前缀开头，甚至可以不加。函数、混写和流程控制没有限制，像写js一样方便。函数甚至有es6才有的参数省略特性！

/* style.styl */
mainColor: #0982c1;
$border: 1px solid #ccc;

/* 流程控制 */
border_and_bg(border_on, border_color, bg_on, bg_color)
  if border_on == true
    border 1px solid border_color
  else
    border 0
  if bg_on == true
    background-color bg_color
  else
    background-color transparent

/* 参数省略 */
padded_box(box_sizing,args...)
  box-sizing box_sizing
  padding args

.callem
    border_and_bg true #000 false #fff
    padded_box border-box 5px 10px

stylus还有很多非常厉害的操作，具体可参考官网：Stylus
