# this和call、apply、bind

# this

this是js中非常重要的关键字。他总是返回一个对象。this的设计目的：在函数体内部，指代函数运行时所在的环境。

# 实质

js之所以有this，跟内存的数据结构有关。原始的对象以字典结构保存，每一个属性名都对应一个属性描述对象，属性的值保存在属性描述对象的value属性里，当属性的值是函数时，引擎会将函数单独保存在一块内存中，然后将该内存的地址赋给该value属性。
由此函数是一个单独的值，他可以在不同的环境中执行。而js允许在函数体内部引用当前环境的其他变量，所以需要有一种机制，能够在函数体内部获得当前的运行环境（context）。this便出现了。

# JS执行环境

执行环境（execution context，运行期上下文）是JS中很重要的一个概念和机制。在JS中，可执行的JS代码分为三种类型，它们分别对应不同类型的执行环境。

  • Global Code：全局的、不在任何函数内的代码
  • Eval Code：使用eval()函数动态执行的JS代码
  • Function Code：用户自定义函数中函数体内JS代码

所有代码都在执行环境中被执行，执行环境定义了变量或函数是否具有访问其他数据的权限，以及在作用域、生存期等方面的处理。全局环境只有一个，且一直存在，而函数环境是进入函数时创建的。当JS代码执行时，就会进入不同的执行环境，每个执行环境由三部分构成：

  • 变量对象（VO, variable object）：变量对象即包含变量的对象，开发者无法直接访问。
  • 作用域属性：[[Scope]]属性是一个指向单向链表的头结点的指针。作用域即变量对象，作用域链是一个由变量对象组成的带头结点的单向链表，作用是用来进行变量查找。 this：指向环境对象

作用域涉及到被调用函数中的变量访问，而且不同调用场景是不一样的。执行环境始终是this关键字的值，它是拥有当前所执行代码的对象的引用。每个执行环境都有一个与之关联的变量对象，环境中定义的所有变量和函数都保存在这个对象中。

# call、apply和bind

call、apply和bind都是定义在函数对象原型上的（Function.prototype）方法。他们的目的都是为了固定/切换this指向。

# call和apply

call和apply很相似，区别在于call的参数传入参数列表，apply的参数传入参数数组。目的是改变函数的调用对象从而改变this指向，并执行该函数。

# 使用场景

  • 当一个对象想要调用一个不属于自己的函数，却不想复制该函数时。
    如：NodeList类数组对象使用数组的 fliter 方法实现过滤；数组使用Math.max查找最大数。
var a = [10, 2, 4, 15, 9];
Math.max.apply(null, a) // 15

# bind

bind在为函数绑定调用对象，返回与这个函数相同功能的新函数。参数就是要绑定的对象。

# 使用场景

  • 绑定函数
var add = function (x, y) {
  return x * this.m + y * this.n;
}

var obj = {
  m: 2,
  n: 2
};

var newAdd = add.bind(obj, 5);
newAdd(5) // 20
  • 改写一些 JavaScript 原生方法的使用形式
var push = Function.prototype.call.bind(Array.prototype.push);
var pop = Function.prototype.call.bind(Array.prototype.pop);

var a = [1 ,2 ,3];
push(a, 4)
a // [1, 2, 3, 4]

pop(a)
a // [1, 2, 3]
